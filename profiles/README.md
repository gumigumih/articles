# Profile Variations

各メディアで使用するプロフィール文を管理するためのメモです。媒体ごとに字数やトーンが異なるため、以下のテンプレートをベースに適宜調整してください。

## 共通要素
- 名前: ぐみ (Megumi)
- 肩書き候補: テクニカルPM / 開発ディレクター / Webエンジニア / プロジェクトディレクター
- 活動領域: Web/アプリ開発の体制づくり、品質改善、運用設計、AI活用
- 強み: マネジメント視点と実装経験の両立、開発プロセス再構成、リリース品質向上
- リンク: note / Zenn / Qiita / GitHub / X(Twitter)

## note 用

### 名前
```
ぐみ@テクニカルPM／開発体制デザイナー
```

### プロフィール文（約140字）
```
Web/アプリ開発の要件定義・UI設計・運用設計まで伴走します。生成AIと既存チームの橋渡しをしながら、開発再構成・リリース品質・ナレッジ整理を回すのが得意です。元Webエンジニア／UIデザイナー→現在テクニカルPM。実装にも触りつつ、続く仕組みづくりを探求中。
```

### ねらい
- noteで好まれる「モットー＋自己開示」の構成を保ちつつ、限られた文字数で専門領域を伝える。
- 「橋渡し」「仕組みづくり」など記事テーマと重なるキーワードを厳選。
- 「テクニカルPM」で技術とマネジメントの橋渡し役を示す。

## Zenn 用（約120字）

### 名前
```
ぐみ@テクニカルPM
```

### プロフィール文
```
テクニカルPMのぐみです。要件定義からUI設計、リリース後の運用まで伴走しながら、AIを活かした開発体制づくりと品質改善に取り組んでいます。現場での工夫や学びを設計思想として整理して発信中。
```

### ねらい
- エンジニア寄りの読者が多いプラットフォームのため、工程横断とAI活用をシンプルに伝える。
- 「伴走」「発信中」など継続的な活動を示す言葉でコミュニティ感を演出。
- note との差別化として、設計思想を共有する姿勢を強調。

## Qiita 用（約100字）

### 名前
```
ぐみ@テクニカルPM／Webエンジニア
```

### プロフィール文
```
テクニカルPM／Webエンジニア。開発体制づくりと業務自動化が好きで、GAS/CLI/AIツールを使った効率化ノウハウを記録しています。Zenn / note でも発信中。
```

### ねらい
- Qiita 利用者が求める「技術的なTips」を想起できるよう、得意領域と具体ツールを列挙。
- 「記録」「発信中」で継続的なアウトプットをアピールし、フォロー動機を作る。
- 他媒体への導線を明記し、横断的な活動を自然に案内。

## GitHub 用（Short Bio）
Technical PM & Web engineer. Reshaping dev teams with AI-assisted processes, release quality, and ops design. Sharing notes on note/Zenn/Qiita.

### ねらい
- 英文で簡潔に役割と専門領域を伝える（AI活用、リリース品質、運用設計）。
- note/Zenn/Qiita への発信を添えて、技術コミュニティの活動を印象づける。

## X (Twitter) 用（160字以内）

### 名前
```
ぐみ@テクニカルPM
```

### プロフィール文
```
テクニカルPMのぐみ。開発体制の再構成、リリース品質向上、AI活用のディレクション。実装もするPM。note・Zenn・Qiitaで現場メモを書いてます。DMでゆるくご相談を。
```

### ねらい
- タイムラインで一目で専門領域が伝わるよう、キーワードを列挙。
- 「実装もするPM」で親近感を、DM歓迎でコンタクトのしやすさを補足。
- 他媒体への導線を簡潔に紹介し、フォロー後の回遊を促す。

## 使い方
- 各メディアの字数やトーンに合わせて調整する。
- 肩書きはテーマに合わせて「テクニカルPM」「開発ディレクター」「Webエンジニア」を使い分ける。
- 重大な役割変更があればここを更新し、各メディアへ反映する。
