# 品質管理・チェックリスト

## 投稿前チェックリスト
- [ ] タイトルが内容を正確に表現している
- [ ] フロントマターが適切に設定されている
- [ ] 見出し構成が論理的
- [ ] コードは実行可能
- [ ] 画像が適切に配置されている
- [ ] リンクが正常に動作する
- [ ] 誤字脱字がない
- [ ] 読者にとって価値がある内容

## 投稿後
- 読者からのフィードバックを確認
- 必要に応じて内容を更新
- 関連記事へのリンクを追加

## 更新ルール
- 内容変更時は更新履歴を記載
- 重要な変更は冒頭に記載
- 古い情報は適切に更新

## 開発運用（コミットメッセージ）
- 形式: `<type>: <summary>`（例: `docs: Zennのtech/idea判定基準を追加`）
- `type` は以下を使用する: `docs`, `feat`, `fix`, `refactor`, `chore`, `test`
- `summary` は日本語で簡潔に記述し、50文字前後を目安にする
- 1コミット1目的を原則にし、本文不要なら件名のみでよい
- 破壊的変更や補足が必要な場合のみ本文を追加する
